# M7 — 통계 + 설정 UI + 멀티 소스 확장

> 대시보드에 통계 화면과 설정 화면(프로필·소스 편집)을 추가한다. 소스를 UI에서 추가/관리할 수 있어 멀티 소스 확장이 가능해진다.

## 만든 것

| 파일 | 역할 |
|------|------|
| `src/db/repository.ts` `getStats()` | GROUP BY 집계: 일자별 수집/소스별/지원단계별/합계 |
| `src/server/index.ts` `GET /api/stats` | 통계 JSON |
| `web/src/components/StatsView.tsx` | 합계 카드 + 가로 막대 3종 |
| `web/src/components/SettingsView.tsx` | 프로필 폼 + 소스 패널(토글·삭제·추가) |
| `web/src/App.tsx` | 통계·설정 탭 추가 |

## 통계 (`GET /api/stats`)

```jsonc
{
  "jobsByDate": [{ "date": "2026-06-19", "count": 3 }],   // 최근 14일
  "jobsBySource": [{ "source": "saramin", "count": 1 }],
  "applicationsByStatus": [{ "status": "interview", "count": 1 }],
  "totals": { "jobs": 3, "applications": 1, "runs": 1 }
}
```
프런트는 합계 카드 + 일자별/소스별/지원단계별 막대로 표시.

## 설정 UI

- **프로필 폼**: 관심 직무·선호/기피 키워드·선호 지역(쉼표 구분), 매칭 임계값, 메일 최대 건수, 받는 메일 → `PUT /api/profile`
- **소스 패널**: 소스 목록(활성 체크박스 토글 `PATCH`, 삭제 `DELETE`) + "HTML 소스 추가" 폼(id·이름·URL·목록/제목/링크/회사 셀렉터 → `POST`). **여기서 새 채용 사이트를 추가**하면 다음 실행부터 수집 대상에 포함된다(멀티 소스 확장).

## 실행

```bash
npm run serve                       # 백엔드 :3000
cd web && npm run dev               # 프런트 :5173
```

## 검증

- 백엔드 `tsc --noEmit` + 프런트 `vite build`(37 모듈) 통과
- `GET /api/stats` 집계 정상 반환
- 스크린샷으로 두 탭 확인:
  - 통계: 합계 3/1/1, 일자별(2026-06-19:3), 소스별(3개 ×1), 지원단계별(면접:1) 막대
  - 설정: 프로필 폼 현재값 채워짐, 소스 테이블(3개, 토글/삭제), 소스 추가 폼
