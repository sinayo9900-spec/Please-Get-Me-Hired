# M2 — LangGraph 파이프라인 (정규화 · 매칭 · 요약)

> 수집된 RawJob을 정규화하고, LLM으로 직무 연관도를 채점·요약한 뒤 DB에 저장하는 **LangGraph StateGraph**를 조립한다.

M2는 4개 하위 단계로 나눠 구현했다: ① normalizer → ② LLM provider → ③ matcher/summarizer → ④ 그래프 조립.

## 만든 것

| 파일 | 역할 |
|------|------|
| `src/agents/normalizer.ts` | `RawJob[]→JobPosting[]`. 멱등 키, 배치 내 중복 제거, `회사명｜직무` 분리. `selectNew()`로 DB 신규만 |
| `src/config/llm_provider.ts` | `LLMProvider` 인터페이스 + `getProvider()` 팩토리(`ENV.LLM_PROVIDER`) |
| `src/llm/cliProvider.ts` | CLI 헤드리스 실행. 프롬프트 stdin 주입, 중립 cwd, 타임아웃. claude는 JSON 엔벨로프 `.result` 파싱 |
| `src/prompts/matcher.ts`, `summarizer.ts` | 자기완결 작업 프롬프트(코딩 에이전트 거부 회피 + JSON/텍스트만 출력) |
| `src/agents/matcher.ts` | 공고별 0~1 채점. 기피 키워드 룰 1차 필터(LLM 절약), 동시성 제한, JSON+Zod, 1회 재시도, 실패 skip. `filterByThreshold()` |
| `src/agents/summarizer.ts` | 통과 공고만 2~3문장 요약 |
| `src/util/pool.ts` | 의존성 없는 동시성 제한기 `mapLimit` |
| `src/graph/state.ts` | 공유 `PipelineState`(Annotation.Root). raw/failedSources/errors는 append 리듀서 |
| `src/graph/pipeline.ts` | StateGraph 배선 + 조건부 엣지 |
| `src/graph/run.ts` | 단일 엔트리(`npm run pipeline`). DB에서 profile/sources 로드, `--limit N` 지원 |

## 그래프 흐름

```
START → collector → (failedSources 있음? → fallback) → normalizer
                  → (없음) ────────────────────────────↗
       → matcher(채점+임계값) → summarizer → persist → END
```

- **collector**: enabled 소스 병렬 수집. 성공분 `raw`, 실패분 `failedSources`.
- **fallback**: playwright 재수집 — **현재 placeholder(M1.5 예정)**. 실패 소스를 오류로만 기록하고 통과.
- **normalizer**: 표준화 + 멱등 키 + 중복 제거 + DB 신규만. `limit` 있으면 상한.
- **matcher**: `LLMProvider`로 채점 → `matchThreshold` 미달 drop.
- **summarizer**: 통과분 요약.
- **persist**: `job_postings` upsert.

## LLM = CLI 구독 (비용 0)

API 대신 로컬 CLI(claude/gemini/codex)를 서브프로세스로 호출(PLAN §6.3). 핵심:

- **프롬프트는 stdin으로만** 주입 → 셸 인젝션 차단 (Windows는 `.cmd` 때문에 `shell:true`지만 외부 텍스트가 args에 없음)
- **중립 cwd(`os.tmpdir()`)** 실행 → 코딩 에이전트가 이 프로젝트 파일/지침을 끌어와 간섭/비용 증가하는 것 방지
- 출력에서 `\{[\s\S]*\}`로 JSON 추출 후 Zod 검증, 실패 시 1회 재시도 후 skip
- `LLM_CONCURRENCY`로 동시 실행 제한, `LLM_TIMEOUT_MS` 타임아웃+kill

## 실행

```bash
npm run pipeline            # 전체 수집→매칭→요약→저장
npm run pipeline -- --limit 3   # 신규 3건만 처리(저비용 드라이런)
```

## 검증

- 단독: matcher 3건 → 0.75 / 0.9 / 0(기피 키워드 룰), 임계값 0.6 통과 2건, 요약 정상(없는 정보 미생성)
- 그래프: `--limit 3` 실행 → 수집 114 → 신규 3 → 임계값 통과 0 → 저장 0, 오류·실패 소스 0, 조건부 엣지 정상 라우팅
- `npx tsc --noEmit` 통과

## 결정 / 메모

- **CLI 코딩 에이전트는 맥락 없는 프롬프트를 인젝션으로 보고 거부**할 수 있음 → 프롬프트를 자기완결 작업으로 프레이밍(검증으로 확인). PLAN §6.3 리스크 실증.
- matcher 기피 키워드 룰 1차 필터로 LLM 호출량 절감.
- `--limit`는 전체 ~114건을 매번 LLM 채점하면 느리고 비싸서 추가한 드라이런/테스트 장치.
- normalizer 멱등 키는 URL 내 가장 긴 4자리+ 숫자(saramin `search_uuid` 같은 휘발성 파라미터 무시).
