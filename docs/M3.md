# M3 — Mailer (일괄 메일 발송 + 발송 로그)

> 매칭·요약된 신규 공고를 하나의 다이제스트 메일로 묶어 발송하고, 결과를 `email_runs`에 기록한다. 그래프의 `persist` 다음 노드로 연결.

## 만든 것

| 파일 | 역할 |
|------|------|
| `src/mail/transport.ts` | ENV SMTP 설정으로 nodemailer transport 생성. `renderDigestHtml()`(순수), `buildDigest()`(정렬·상한·메시지 구성) |
| `src/agents/mailer.ts` | `sendDigest()` — 발송 + `email_runs` 기록. 0건 skip, 발송 실패는 비치명적 |
| `src/graph/pipeline.ts` | `mailerNode` 추가, `persist → mailer → END` 배선 |
| `src/graph/state.ts` | `emailResult` 채널 추가 |

## 동작

1. `summarized` 공고를 `matchScore` 내림차순 정렬 → `profile.email.maxItems` 상한 적용(초과분은 "그 외 N건" 안내)
2. HTML 다이제스트 렌더(회사·제목·적합도·추천 사유·요약·지원 링크, 모두 HTML escape)
3. `from = ENV.MAIL_FROM`, `to = profile.email.to`, 제목 `[채용 알림] {날짜} 신규 매칭 N건`
4. 발송 후 `email_runs`에 `{runDate, jobCount, success}` 기록

## 멱등성 / 재발송 방지

normalizer가 이미 **DB 신규 공고만** 통과시키므로(`selectNew`), 매 실행 시 새로 매칭된 공고만 메일에 담긴다. 같은 공고가 다음 날 다시 발송되지 않는다.

## 실패 내성

- 공고 0건 → 발송하지 않고 skip(로그도 남기지 않음)
- SMTP 발송 실패 → `email_runs`에 `success=false, error` 기록하고 파이프라인은 계속 진행(메일 실패가 전체를 막지 않음)

## 실행 / 설정

```bash
npm run pipeline    # … → persist → mailer
```

실제 발송하려면 `.env`에 SMTP 설정 필요(Gmail은 **앱 비밀번호**):

```
SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_USER=you@gmail.com
SMTP_PASS=<앱 비밀번호>
MAIL_FROM="Please-Get-Me-Hired <you@gmail.com>"
```

## 검증

- `sendDigest` (nodemailer `jsonTransport`, 네트워크 없음): 3건 + maxItems=2 → 발송 count 2(상한 적용) ✓
- 0건 → `skipped` ✓
- `email_runs` 기록 확인(date·jobCount·success) ✓
- 그래프 `--limit 0` 실행 → mailer 노드 도달 후 "발송 안 함(0건)" 정상 ✓
- `npx tsc --noEmit` 통과

## 메모

- 실제 Gmail 발송 검증은 앱 비밀번호가 필요해 보류(드라이 검증으로 대체). 자격증명 넣으면 바로 발송됨.
- 대시보드 링크 자리: 대시보드(M5~6) 완성 후 푸터에 추가 예정.
