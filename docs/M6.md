# M6 — 웹 대시보드 (Vite + React + TS)

> M5 API를 소비하는 프런트. 오늘의 공고 열람 + 지원 현황 칸반보드 + 수동 실행.

## 만든 것 (`web/`)

| 파일 | 역할 |
|------|------|
| `src/api.ts` | fetch 래퍼. dev에선 vite 프록시(`/api`→:3000), 배포 시 `VITE_API_BASE` |
| `src/types.ts` | 백엔드와 맞춘 도메인 타입 + 상태 라벨 |
| `src/App.tsx` | 헤더(프로필·"지금 실행") + 탭(공고/지원현황) |
| `src/components/JobsView.tsx` | 공고 목록, 최소 적합도 필터, "지원 목록에 추가" |
| `src/components/ApplicationsBoard.tsx` | 7개 상태 칸반, 카드별 상태 `<select>`로 변경 |
| `src/styles.css` | 경량 스타일 |
| `vite.config.ts` | dev 서버 + `/api` 프록시 |

## 기능

- **오늘의 공고**: `GET /api/jobs`(최소 적합도 필터). 카드에 회사·지역·적합도·추천사유·요약, 링크. "지원 목록에 추가" → `POST /api/applications`(status=bookmarked)
- **지원 현황 보드**: `GET /api/applications` + `/api/jobs`(제목 매핑). 관심→지원→과제→면접→오퍼/불합격/종료 컬럼. 카드 `<select>`로 상태 변경 → `PATCH /api/applications/:id`
- **지금 실행**: `POST /api/run`으로 파이프라인 수동 트리거
- 헤더에 프로필 관심 직무·임계값 표시

## 실행

```bash
# 1) 백엔드
npm run serve            # :3000
# 2) 프런트 (web/)
cd web && npm install && npm run dev   # :5173 (/api는 :3000으로 프록시)
```

배포 빌드: `cd web && npm run build` → `web/dist/`.

## 검증

- `tsc --noEmit` + `vite build` 통과(35 모듈 번들)
- 백엔드+dev 서버 실행 후 스크린샷으로 양 탭 확인:
  - 공고 탭: 수집 공고 3건 렌더(회사·제목·링크·추가 버튼)
  - 지원 탭: 칸반 렌더, saramin 지원건이 **면접** 컬럼에 표시, 상태 드롭다운 동작

## 메모

- 칸반 상태 변경은 드래그 대신 `<select>`(견고·단순). 드래그앤드롭은 추후 개선.
- 인증 없음(로컬 단일 사용자). dev DB의 프로필 한글이 깨졌던 건 curl(git bash) 인코딩 문제였고 ts-node로 복구함 — 앱/저장 데이터는 UTF-8 정상.
- 통계·설정 화면은 PLAN의 2차(M7) 항목으로 남김.
