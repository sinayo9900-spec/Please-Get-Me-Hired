# M0 — Foundation (설정 · DB · 시드)

> 본격 파이프라인 전, 모든 단계가 공유할 **설정 스키마 · env 로딩 · DB 레이어**를 깐다.

## 범위

- 설정/도메인 타입을 Zod 스키마로 정의
- `.env`를 검증해 타입 안전한 상수로 노출
- SQLite 호환 DB 스키마 + repository(CRUD)
- 마이그레이션·시드 스크립트

## 만든 것

| 파일 | 역할 |
|------|------|
| `src/config/schema.ts` | Zod 스키마 + 추론 타입: `Profile`, `Source`, `JobPosting`, `Application`, `EmailRun`. DB의 JSON(TEXT) 컬럼을 로드 시 검증 |
| `src/config/constant.ts` | `.env`를 Zod로 검증해 `ENV`로 export. 누락/형식 오류 시 즉시 throw. 앱은 `process.env`를 직접 읽지 않고 이 모듈만 참조 |
| `src/db/schema.sql` | 테이블: `profile`, `sources`, `job_postings`, `applications`, `email_runs` (+ 인덱스). 불리언은 0/1, 가변 구조는 JSON TEXT |
| `src/db/client.ts` | libSQL 클라이언트 + `migrate()` (schema.sql 실행, `IF NOT EXISTS`로 반복 안전). 로컬 파일 DB 디렉터리 자동 생성 |
| `src/db/repository.ts` | CRUD. Row↔도메인 매핑 + 반환 시 Zod 검증. 멱등성용 `findExistingJobIds(ids)` 포함 |
| `src/db/migrate.ts` | 테이블 생성 엔트리 |
| `src/db/seed.ts` | 기본 `profile` 1건 + (초기) 예시 source 시드 (멱등) |

## 실행

```bash
npm run migrate   # 테이블 생성
npm run seed      # 기본 profile 시드
npm run typecheck # tsc --noEmit
```

`.env`는 `.env.example`을 복사해 작성. 필수: `SMTP_USER`, `SMTP_PASS`, `MAIL_FROM`.

## 검증

- `npx tsc --noEmit` 통과
- `npm run seed` 후 재실행 시 멱등(이미 존재 → 건너뜀)
- repository 라운드트립: JSON 컬럼·boolean·중첩 객체 모두 Zod 검증 통과

## 결정

- **DB 드라이버: `better-sqlite3` → `@libsql/client`.** 개발 머신에 VS C++ 빌드 도구가 없어 better-sqlite3 네이티브 컴파일이 실패. libSQL은 사전 빌드 바이너리라 컴파일러 불필요. (PLAN §2·§13)
- **Zod v4** 설치됨 → `z.record(key, value)` 2-인자 형태 사용.
- 설정 파일을 두지 않고 `profile`/`sources`를 **DB로 관리** (PLAN §5).
