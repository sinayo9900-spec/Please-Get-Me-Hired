# M5 — 대시보드 백엔드 API (express)

> DB와 파이프라인 위에 REST API를 올린다. 프런트(M6)와 수동 운영의 진입점.

## 만든 것

| 파일 | 역할 |
|------|------|
| `src/server/index.ts` | express 앱(`createServer()`) + 라우트. `npm run serve` (포트 `ENV.DASHBOARD_PORT`, 기본 3000) |

## 엔드포인트

| 메서드 · 경로 | 설명 |
|---|---|
| `GET /api/health` | 헬스체크 |
| `GET /api/jobs?date=&minScore=&source=` | 수집 공고 목록/필터 |
| `GET /api/jobs/:id` | 공고 단건 |
| `GET /api/profile` · `PUT /api/profile` | 직무/필터·메일 설정 조회·수정 |
| `GET /api/sources` · `POST` · `PATCH /:id` · `DELETE /:id` | 수집 소스 관리(활성 토글 포함) |
| `GET /api/applications?status=` · `POST` · `PATCH /:id` | 지원 현황 CRUD |
| `GET /api/runs` | 메일 발송 이력 |
| `POST /api/run` | 파이프라인 수동 실행(백그라운드, 202) |

## 설계 포인트

- **검증**: 생성/수정은 Zod 스키마로 검증, 실패 시 `400` + 이슈 목록.
- **PATCH 병합**: 기존 row를 읽어 body를 덮어쓴 뒤 재검증·upsert(부분 수정).
- **POST /api/run**: 파이프라인이 수 분 걸릴 수 있어 응답을 막지 않고 백그라운드 실행 후 `202 {started:true}`.
- **CORS**: 프런트(별도 origin)용 허용 헤더 + OPTIONS 처리.
- **에러**: 비동기 핸들러 예외를 `wrap()`으로 모아 `500` 핸들러로.
- 서버 기동은 `require.main === module` 가드 — 테스트/임포트 시 listen 안 함.

## 실행

```bash
npm run serve     # http://localhost:3000
```

## 검증 (라이브, 포트 3000)

- `GET` health / profile / sources(3건 enabled) / jobs / runs 정상
- 지원 현황: `POST`(생성) → `PATCH`(applied→interview) → `GET` 반영 확인
- `PUT /api/profile` 수정 반영
- 잘못된 status로 `POST` → `400`
- `npx tsc --noEmit` 통과

## 메모

- Express 5라 `req.params.id`가 `string | string[]` 타입 → `String()`으로 정규화.
- 한글이 curl 콘솔에 깨져 보일 수 있으나 저장/응답 데이터는 UTF-8 정상(콘솔 코드페이지 문제).
- 인증 없음(단일 사용자 로컬 전제). 외부 노출 시 인증 추가 필요.
