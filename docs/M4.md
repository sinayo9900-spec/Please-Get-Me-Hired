# M4 — 스케줄러 (매일 자동 실행)

> `npm run pipeline`을 매일 정해진 시각에 자동 실행한다. CLI 구독 LLM을 쓰므로 해당 CLI가 로그인된 **로컬/상주 머신**에서 돌려야 한다(PLAN §8).

## 만든 것

| 파일 | 역할 |
|------|------|
| `src/graph/run.ts` | `runPipeline({limit?})` 분리 — cron·CLI·대시보드 공유 진입점. CLI 직접 실행은 `require.main === module` 가드 |
| `src/scheduler/cron.ts` | node-cron 상주 스케줄러 |
| `src/config/constant.ts` | `SCHEDULE_CRON`(기본 `0 8 * * *`), `SCHEDULE_TZ`(기본 `Asia/Seoul`) 추가 |

## 동작

- `ENV.SCHEDULE_CRON` 주기마다 `runPipeline()` 실행 (`timezone = ENV.SCHEDULE_TZ`)
- 시작 시 cron 식 유효성 검사(`cron.validate`) — 잘못되면 즉시 종료
- **중복 실행 방지**: 이전 실행이 끝나지 않았으면 해당 주기 건너뜀
- per-run 오류는 잡아서 로그만 — 스케줄러는 계속 동작
- `--run-now`: 시작 즉시 1회 실행(수동 트리거/점검용)

## 실행 방식 2가지

### A. 상주 프로세스 (node-cron)
```bash
npm run schedule            # 매일 08:00 KST 실행, Ctrl+C로 종료
npm run schedule -- --run-now   # 시작과 동시에 1회 실행
```
시각/시간대 변경은 `.env`:
```
SCHEDULE_CRON=0 8 * * *
SCHEDULE_TZ=Asia/Seoul
```
> 머신 재부팅 후 자동 기동하려면 OS 서비스(PM2, NSSM 등)로 등록.

### B. Windows 작업 스케줄러 (상주 불필요)
매일 08:00에 `npm run pipeline`을 한 번 실행하도록 등록:
```powershell
schtasks /Create /SC DAILY /ST 08:00 /TN "PleaseGetMeHired" ^
  /TR "cmd /c cd /d D:\Project\병아리\Please-Get-Me-Hired && npm run pipeline"
```
> 상주 프로세스가 필요 없어 개인 PC에 적합. 단, 실행 시점에 PC가 켜져 있어야 함.

## 검증

- 스케줄러 시작 → 기본 cron 등록 로그 출력, 즉시 발화 없음 ✓
- 잘못된 `SCHEDULE_CRON` → 거부 후 종료 ✓
- `npx tsc --noEmit` 통과
- (`--run-now` 전체 실행은 LLM 비용 때문에 보류 — `runPipeline` 자체는 M2/M3에서 검증됨)
