# M1 — Collector 어댑터 + 라이브 소스 연동

> 설정·DB(M0) 위에, 실제 채용 사이트에서 공고를 긁어오는 **수집기**를 만든다.

## 범위

- 전략 패턴 수집기(api/html 어댑터) + 디스패처
- 소스별 실패 격리 (한 사이트 실패가 전체를 막지 않음)
- 실제 3개 사이트를 DB 소스로 등록하고 라이브 수집 확인

## 만든 것

| 파일 | 역할 |
|------|------|
| `src/collectors/types.ts` | `RawJob`(정규화 전 결과), `CollectorAdapter` 인터페이스, 브라우저 위장 기본 헤더 |
| `src/collectors/htmlAdapter.ts` | cheerio 스크래퍼. `Source.select.list`로 항목 순회, `fields`로 추출 |
| `src/collectors/apiAdapter.ts` | JSON 응답을 점 경로(`a.b.0.c`)로 추출 |
| `src/collectors/index.ts` | `collect(source)` / `collectAll(sources)` 디스패처. 예외를 `CollectResult`로 감싸 소스별 성공/실패 분리 |
| `src/collectors/run.ts` | DB의 `enabled` 소스를 수집해 요약 출력 (`npm run collect`) |
| `src/db/seed-sources.ts` | 실제 소스 3개 upsert + M0 placeholder 제거 (`npm run seed:sources`) |

## 필드 셀렉터 문법 (html)

`Source.select.fields`의 값은 cheerio 셀렉터이며:

- `".jd-card-title a"` → 해당 요소의 **텍스트**
- `".jd-card-title a@href"` → 해당 요소의 **속성값** (`@속성`)
- 상대 URL은 소스 origin 기준으로 절대화

알려진 필드 키만 매핑: `title`, `company`, `url`, `location`, `employmentType`, `description`, `postedAt`. `title`/`url`이 비면 그 항목은 스킵.

## 연동한 소스

| id | 사이트 | list | title / url | company / location |
|----|--------|------|-------------|--------------------|
| `peoplenjob` | 피플앤잡 | `.jd-card` | `.jd-card-title a` / `…a@href` | `.jd-card-company` / `.jd-card-meta-location-text` |
| `saramin` | 사람인(데이터 엔지니어) | `.item_recruit` | `.job_tit a` / `…a@href`(상대) | `.corp_name a` / `.job_condition span` |
| `inthiswork` | 인디스워크 | `.sub-entry` | `.dpt-title-link` / `…@href` | (제목에 `회사｜직무`) |

## 실행

```bash
npm run seed:sources   # 3개 소스 DB 등록
npm run collect        # 수집 실행 + 요약 출력
```

## 검증 (라이브)

```
✓ peoplenjob: 30건
✓ saramin: 40건
✓ inthiswork: 44건
총 114건 수집. 실패 소스: 없음
```

- 상대 URL 절대화 정상 (saramin `/zf_user/...` → 전체 URL)
- 회사·지역 추출 정상
- 세 사이트 모두 정적 HTML로 수집 가능 (JS 렌더링 fallback 불필요)

## 결정 / 메모

- **inthiswork 제목이 `회사명｜직무` 합쳐진 형태** → 회사/직무 분리는 **M2 normalizer**로 미룸.
- saramin·inthiswork URL의 `page`/`paged1`이 1페이지 고정. 다중 페이지 수집은 필요 시 추가.
- 세 사이트 모두 정적 수집 성공이라 **M1.5(Playwright fallback)는 당장 불필요** — 추후 막히는 사이트가 생기면 구현.
- HTML 구조 분석은 cheerio로 1회 받아 셀렉터 도출(스크래핑은 사이트 약관·rate limit 유의).
